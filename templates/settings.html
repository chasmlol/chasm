Generation sampling

Forwarded verbatim to llama.cpp's /v1/chat/completions on every turn (read fresh per request). Defaults match prior behaviour; off-values (top-k/min-p 0, max tokens 0, seed -1) are omitted from the request.

{{ page.llm_sampling.temperature }}

Higher = more varied/creative; lower = more focused and deterministic.

{{ page.llm_sampling.top_p }}
{{ page.llm_sampling.min_p }}

0 disables min-p (the default).

{{ page.llm_sampling.repeat_penalty }}

1.0 = off. Above 1.0 discourages repeating tokens.

Top-k 0 = disabled. Max tokens 0 = no cap (server decides). n_ctx 0 = use the model's loaded context. Seed -1 = random each call.

Live chat orchestrator

In a scene with two or more present, audible NPCs, a small "director" call decides who speaks next and in what order. With one NPC (or when off) the single present NPC just speaks — no extra call.

The director sees only the list of present characters and the recent conversation. Leave the prompt blank to reset it to the default.
